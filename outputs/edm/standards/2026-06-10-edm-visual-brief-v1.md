# EDM 视觉规范 Brief v1（Ocoopa / Diveblues）

> 给设计师执行的 EDM 视觉 spec。内部对齐中文；最终邮件内文案按品牌需要输出英文。

## 1. 文件定位

本文件定义 Ocoopa / Diveblues 第一版 EDM 视觉系统方向，用于 2026.06-09 EDM 第一阶段“标准统一与基础重构”。

它解决四件事：

1. 从老旧电商版式切换到生活方式 / 编辑化 EDM 视觉。
2. 建立可复用模块，让后续新品、促销、内容、互动邮件能快速生产。
3. 保留 Ocoopa 与 Diveblues 的品牌色温、影调、人物状态差异。
4. 让设计交付能同时服务转化和品牌 / 留存，而不是只做 banner。

本文件不是 HTML 模板，不替代设计稿，不规定平台选型。当前平台背景为 Shopify / 独立站，后续是否接 Klaviyo 等工具另行评估。

## 2. Scope & Ownership 边界

### 2.1 DTC 是 v1 最高优先级

EDM v1 的视觉规范优先服务 DTC 用户邮件，包括官网订阅、Shopify 转化链路、自动化流和 DTC 品牌内容。

B2B 不作为 v1 的独立视觉轨道，不做招商邮件视觉规范。

### 2.2 B2B 素材只能作为 DTC 品牌可信度内容

展会、陈列、零售露出、渠道现场可以作为 DTC 邮件中的“品牌可信度”视觉素材出现。

可用方向：

- 展会现场细节：用于说明品牌被真实展示、被行业看见。
- 零售陈列 / 货架露出：用于增强购买安心和品牌存在感。
- 线下活动图：用于品牌 behind-the-scenes 或 community recap。

不可用方向：

- 不做招商海报风。
- 不做经销政策、代理招募、批发权益视觉。
- 不把 DTC 邮件设计成 B2B sales deck。

### 2.3 独立 B2B EDM 另立轨道

如后续需要独立 B2B EDM，应作为 Pan 拥有的独立轨道。EDM 专项可提供创意支持、视觉口径、版式建议和素材适配，但不拥有 B2B 线索结果。

### 2.4 EDM 专项只对 EDM 自身指标负责

视觉规范服务的指标包括：

- Open Rate（辅助信号）
- Unsubscribe Rate
- CTR
- Conversion
- RPE
- Subscription Growth

EDM 专项不对 B2B 线索数量、招商结果、经销谈判或商超入驻结果负责。

## 3. 视觉总原则

### 3.1 从电商模板转向品牌内容版式

当前要规避的老版式是：

```text
大图 -> 文案 -> 按钮 -> 商品堆叠
```

v1 应转向：

```text
场景第一眼 -> 品牌语气标题 -> 内容 / 产品价值 -> 真实证据 -> 清楚 CTA
```

视觉不是把商品图排满，而是让用户在第一屏先感到“这封邮件属于这个品牌”。

### 3.2 移动端优先

方法论基准提示移动端占邮件打开 55%+。因此所有 EDM 视觉必须先按移动端检验：

- 第一屏主题能否在 3 秒内看懂。
- 主图裁切是否保留核心人物 / 产品 / 场景。
- 标题是否短到可读。
- CTA 是否足够清楚、可点击。
- 多图拼贴是否在小屏仍能看出重点。

### 3.3 模块化，而不是每次重做

v1 建议沉淀以下模块：

| 模块 | 用途 | 设计要求 |
|---|---|---|
| Hero scene | 第一屏进入品牌 / 场景 | 大图或强视觉，但不做硬广棚拍 |
| Editorial intro | 品牌价值 / 内容邮件开场 | 留白充足，标题短，正文少 |
| Product detail | 展示产品特征 | 微距、手部、材质、使用细节 |
| UGC / Review | 可信度与社区感 | 不做密集评价墙；与场景图交错 |
| Offer / Promo | 转化信息 | 折扣 / 活动清楚，但不压倒品牌视觉 |
| Credibility block | 展会 / 零售 / 陈列露出 | 作为品牌被看见的证据，不写招商 |
| CTA band | 下一步动作 | 一个主动作，按钮样式稳定 |
| Footer / Social | 留存和社媒延展 | 清楚、轻，不抢主 CTA |

## 4. 邮件类型与视觉配比

视觉系统要配合每周邮件类型，而不是所有邮件都长成促销页。

| 类型 | 周节奏 | 视觉任务 | 指标导向 |
|---|---:|---|---|
| 促销 / 转化邮件 | 每周 1 封 | 清楚呈现 offer、商品、CTA；仍保留品牌场景 | RPE、Conversion、CTR |
| 品牌价值 / 内容邮件 | 每周 1 封 | 生活方式、编辑化、UGC、behind-the-scenes、品牌可信度 | CTR、低退订、订阅增长、名单健康 |
| 互动 / 社区邮件 | 每周 0-1 封 | 投票、quiz、UGC 征集、review 邀请，视觉轻快直接 | CTR、互动点击、UGC / review 行动 |

促销邮件可以更强 CTA，但品牌价值邮件和互动邮件必须让用户有继续订阅的理由。视觉复盘时不能只用短期 RPE 判断所有类型。

## 5. 版式方向

### 5.1 推荐结构

#### A. Scene-led Hero

适用：品牌内容、新品、季节主题、Ocoopa 温暖场景、Diveblues 动态场景。

结构：

1. 真实生活场景主图。
2. 1 行短标题。
3. 1 段短说明。
4. 主 CTA。

#### B. Editorial Stack

适用：Rare Beauty 式长文 / 编辑化启发、品牌观点、场景教育。

结构：

1. 标题 + 短导语。
2. 1-3 个内容段落或图文模块。
3. UGC / review / 产品细节作穿插。
4. 低压 CTA。

#### C. Product-in-Life Grid

适用：新品、颜色、系列、使用方式。

结构：

1. 生活场景图。
2. 产品局部图。
3. 用户动作图。
4. 简短说明与 CTA。

#### D. UGC / Community Strip

适用：UGC、好评、社媒内容延展。

结构：

1. 2-4 个真实使用片段。
2. 1 条 review 或用户短句。
3. CTA：看更多 / 分享你的 / 选你的。

#### E. Credibility Block

适用：展会、陈列、零售露出进入 DTC 邮件。

结构：

1. 现场或陈列细节图。
2. 简短说明“where the brand showed up”。
3. 连接回 DTC 用户价值：购买安心、品牌存在感、生活方式扩展。
4. 不放招商 CTA。

### 5.2 留白与信息密度

留白是为了让内容可读和品牌更有秩序，不是为了空洞高级感。

使用原则：

- 每屏只承担一个主要任务。
- 标题、图片、CTA 之间留出呼吸感。
- 如果信息很多，拆成模块，不要塞进一张长 banner。
- 产品图、UGC、review、CTA 要有层级，不要平铺。

Rhode 的极简留白可作为整体视觉方向参考；具体邮件结构没有直接 teardown 时，不写成确定事实。

## 6. Ocoopa 视觉规范

### 6.1 视觉关键词

- Warm
- Bright
- Sunny
- Clean
- High-saturation
- Film feel
- Relaxed real life
- Tacit friend / family perspective

### 6.2 影调

Ocoopa 的画面要让用户同时感到两件事：

1. 环境里确实有“需要温暖”的瞬间。
2. 人因为得到了温暖而松弛下来。

执行方向：

- 明暗关系清晰，避免灰、脏、糊。
- 可以有阳光、暖色块、高饱和色彩碰撞、胶片质感。
- 人物动作应自然，像真实生活中的下意识反应。
- 产品应像已经属于这个生活瞬间，而不是被硬摆进去。

### 6.3 场景建议

- red-eye flight / airport lounge
- early commute
- cold office
- winter walk
- waiting outside
- gifting table / unboxing
- family / friend moments
- hands, pockets, bag, desk, coat, travel tray

### 6.4 图片选择

优先：

- 人不直盯镜头。
- 手部、触感、拿起、放入口袋、递给对方。
- 暖光、清晰色块、轻微生活凌乱但不脏。
- 能感到“松弛”和“被照顾”的动作。

避免：

- 棚拍式硬展示。
- 模特面对镜头摆出修饰性姿势。
- 极端雪地 / 生存装备感成为唯一场景。
- 过冷、过灰、过技术化的画面。

## 7. Diveblues 视觉规范

### 7.1 视觉关键词

- Cool
- Clean
- Bright
- Pure
- Moving
- Real
- Multi-scene
- Multi-role
- Participant perspective

### 7.2 影调

Diveblues 的画面要让用户感到清凉、干净、明亮，并且人和产品正在参与生活。

执行方向：

- 冷色 / 清亮色作为主感受。
- 避免大面积暖色造成燥热。
- 画面需要有行动、移动、出门、转场、互动。
- 人物状态要真实、多变、多面，有活人感。

### 7.3 场景建议

- travel
- commute
- out-and-about
- family day
- friends
- city walk
- packing / carrying
- desk-to-street transition
- parent / child / colleague / friend roles

### 7.4 图片选择

优先：

- 正在走、拿、放、交谈、出门、移动。
- 清爽背景、干净日光、蓝 / 白 / 清亮中性色。
- 多角色生活状态：上班、朋友、家庭、旅行。
- 产品在生活动作中被自然带着走。

避免：

- 大面积暖色、橙黄、燥热感。
- 灰暗、杂乱、焦虑。
- 静态产品摆拍作为主视觉。
- 过度精修到没有真实生活感。

## 8. UGC / Review 使用规范

UGC 的作用是增加真实感、社区感和信任，不是把截图贴满邮件。

### 8.1 可用 UGC

- 真实用户使用场景。
- 真实评价短句。
- 社媒截图或用户图，但要确认授权。
- 好评可视化，但文字要少。
- 对比“用户怎么用”而不是“品牌怎么说”。

### 8.2 Ocoopa UGC 方向

- 手部温暖反应。
- 旅行等待、通勤、办公室、户外短停留。
- 礼赠、家人朋友互送。
- 产品被自然握住、放在桌上、放进口袋、随身带着。

### 8.3 Diveblues UGC 方向

- 携带、移动、出街、旅行、通勤。
- 朋友 / 家庭 / 同事多角色互动。
- 产品与用户当天状态有关，而不是静物装饰。
- 清凉、干净、明亮的真实场景。

### 8.4 避免项

- 密集评价墙。
- 截图过小导致移动端不可读。
- 没有授权的用户内容。
- 只选最精修、最像广告的“伪 UGC”。
- 两品牌共用同一批图，只换 logo。

## 9. 展会 / 陈列 / 零售露出的视觉用法

这些素材只能作为 DTC 品牌可信度模块，不做招商。

### 9.1 可用模块标题方向

英文方向示例：

- `Seen in real life`
- `Where the brand showed up`
- `A closer look from the floor`
- `From display to everyday use`

### 9.2 可用画面

- 品牌 booth 局部。
- 产品陈列细节。
- 零售露出中用户可理解的现场图。
- 团队 behind-the-scenes。

### 9.3 不可用画面

- 渠道政策、招商权益、批发表格。
- B2B sales deck 截图。
- 大量商务合影，没有 DTC 用户可理解价值。
- 看起来像“找代理”的 CTA 区块。

## 10. CTA 视觉规范

### 10.1 层级

- 主 CTA：一个，视觉最明确。
- 次 CTA：最多一个，弱于主 CTA。
- 文字链接：只用于内容延伸，不抢主转化。

### 10.2 样式方向

Ocoopa：

- 按钮可更温暖、明亮、亲近。
- 与画面暖色 / 高饱和色块协调。
- 避免压迫式大红促销按钮。

Diveblues：

- 按钮应清爽、干净、轻。
- 可使用冷色、白色、清亮中性色。
- 避免厚重暖色按钮成为视觉重心。

## 11. 设计避坑

- 不要把邮件做成商品详情页截图。
- 不要把所有产品卖点塞进第一屏。
- 不要一封邮件出现多个同级 CTA。
- 不要让折扣条压过品牌主图。
- 不要用没有生活状态的产品图承担整封邮件。
- 不要使用明显摆拍、镜头感太强的人物。
- 不要两个品牌共用同一视觉，只改 logo。
- 不要无来源地照搬对标品牌具体结构；无法追溯到 URL 的 benchmark 细节标 `to-source`。
- 不要把 DTC 邮件做成招商物料。

## 12. 交付检查清单

设计交付前检查：

- 这封邮件的类型是促销、品牌价值、互动还是自动化？
- 第一屏是否一眼可懂？
- 移动端主图裁切是否保留重点？
- CTA 是否只有一个主动作？
- 产品是否嵌入生活场景，而非孤立摆拍？
- UGC / review 是否可读、可信、有授权？
- Ocoopa 是否足够暖、阳光、松弛？
- Diveblues 是否足够凉、干净、明亮、动态？
- 展会 / 陈列素材是否只是品牌可信度，不是招商？
- 是否有未确认数字、未经授权素材或 to-source claim？
