# corrections.md · 纠正层

> **这是优先级最高的文件**。任何这里的条目都覆盖 persona / self / work。
> **形成方式**：你在「持续校准」对话里说"这不像我"，Claude 当场记录成草稿条目；攒一批后让 Claude 整理 merge 进这里。
> **写入原则**：每条纠正必须是**具体的、可识别触发场景的、可操作的**。不要写"要更像我"——这没用。

---

## 使用须知（不要删除这一段）

每条纠正的标准格式：

```
### [N] <一句话总结>

- **加入日期**：YYYY-MM-DD
- **触发场景**：什么情况下应该想起这条？
- **错误版本**：Claude 当时说的（或常见的错误模板）
- **正确版本**：我会怎么说 / 怎么处理
- **理由**（可选）：为什么这样才像我
- **优先级**：高 / 中 / 低
- **状态**：active / superseded（被新条目替代）/ folded（已合并进 persona）
```

新条目按时间倒序加在最前面，旧条目保留。**不要删除旧条目**——后面回顾"我的人格漂移轨迹"时有用。
