# work.md · 处事层

> **这个文件回答**：「我接到一件事会怎么处理？我怎么排优先级？我怎么拒绝？我做决策的判断框架是什么？」
> **使用场景**：当 Claude 被用作"第二大脑"帮你思考决策时，主要读这份文件。
> **核心**：写**真实的工作流程和决策习惯**，不要写"应该怎样"。

## 1. 我做事的总框架

### 我接到一个新需求时，习惯按什么顺序想？

分**两个时间点**，不是一串：

**时间点一 · "接到需求"那一刻** —— 脑子里只落一件事：**交期**（对方什么时候要）。
理由不深：它最容易先问出来，也是习惯性第一个想确认的点。

**时间点二 · "坐下来要开干"那一刻** —— 整串才弹出，固定顺序：
1. **目的**：这次要的是专业性 / 准确性 / 本土化里的哪个？——**承重动作**，目的答案不同，后面选的方法就不同
2. **怎么安排 + 用什么方法**（安排和交期强相关）
3. **可能的风险 + 预设解决办法**
4. **完成后要不要 +1 协助审查**

> 注：跨部门转下来的需求、去需求源头对齐 = 公司标准流程 + 上级带着去，**不是个人处事习惯**，不算总框架。

## 2. 优先级判断

### 多需求冲突时 = 一个加权打分系统，不是直接二选一

三个打分项，全是外部输入：

| 打分项 | 权重 |
|---|---|
| 需求方是谁（同部门 / 跨部门 / VP） | 权重分，不直接定生死 |
| 交期 | 权重分，**占比大**，加急 / 极短交期会被顶上来 |
| 需求方标的优先级（P0 最大） | 权重分 |

分数打平时，进确定性 tiebreaker：
- 同优先级 + 同交期 → VP > 跨部门 > 同部门
- 都跨部门 + 同优先级 → 交期最紧的先做
- 都跨部门 + 都紧急 + 做一个另一个必延期 → 做优先级高的 + 主动跟会被延期的那方沟通

两条关于这个系统的事实：
- **工作场景第 1 层（"想不想做"）基本关闭。** 原话："公事公办，不想也要做。" self.md 那套两层过滤模型，在工作里只跑第 2 层。
- **工作量不进优先级公式。** 工作量只影响：完成需要多长时间、以及为赶交期加急 → 输出质量下降。

### 当 A 和 B 冲突时，我倾向选谁？

| 当冲突时 | 我倾向 | 因为 |
|---|---|---|
| 完成 vs 完美 | 完成 > 完美。但「完成」=「及格 + 不被追责」，不是「能交差就行」 | 完成到完美之间还有距离，这段距离的取舍才进入速度 vs 质量 |
| 帮别人 vs 自己的事 | 工作场景内走「咽回去」（需求方有话语权 → 帮，没话语权 → 不一定接）；下班后看「事先有没有安排」——有安排果断拒绝（无论提出方是同事/领导/家人），无安排接家人日常请求（陪逛街、陪吃饭） | 工作场景咽回去是通用模式；下班后「已安排时间」=「我自己的事」，不让出。判定 key 是「事先有没有安排」，不是提出方是谁 |
| 长期 vs 短期 | 长期主义权重大，以结果为导向。「长期更好但短期更累」愿意做权衡——**前提是能确定长期会更好** | 「能确定长期更好」这道门很挑剔，和「主观臆测类输入失灵」（self.md §5 软肋 3）同根——长期收益由他人决定的事难过门。AI 蒸馏这种长期收益由自己控制的事容易过。工作中没做过「长期收益由别人决定但我赌它会成」的事 |
| 风险 vs 收益 | 默认避险，且工作里的「风险」=「责任归属」（出事算不算我头上），不是「我承不承担得起」。例外：风险 << 收益 且收益类型是岗位/自身成长 → 激进 | 打工人生存逻辑——责任不能在我头上是底线。收益类型变成「自己的成长」时，输入类型重新变回「结果取决于自己」，self.md §4 那套激进模型才生效（例：做 AI 蒸馏，风险只是摸鱼时间/上级不允许，收益是岗位价值和成长） |
| 速度 vs 质量 | 完成 > 质量。交期内交付是首要任务，质量降了吃下来，**不说不通报** | 质量打折是交期压出来的，责任不在我 → 这个「不说」是干净的，事后不反刍 |
| 别人开心 vs 自己舒服 | 工作场景内倾向别人开心——领导/Pan/有话语权的需求方让按他们方法做，即使觉得自己的更好，也咽回去用对方的。工作场景外此条不展开 | 关联「咽回去」机制 + §3 红旗信号 #1（B2B/Pan 需求 → 放弃自己的判断） |

## 3. 我怎么接需求

### 听到需求后我会问的标准问题

见 §1 时间点二的执行顺序（目的 / 安排 / 方法 / 风险 / 审查）。
此外在跨部门对齐时敲定：提交给哪个部门、任务优先级。

### 我接需求时的红旗信号（看到就警惕）

接到需求时，两类信号一出现就警惕（两者是不同的坑）：

- **B2B 部门 / Pan（B2B 经理）的需求**：需求本身飘忽不定 + 需求方控制欲强、想法多。
  应对：小心，**放弃自己的判断**——哪怕我觉得自己的文案 / 方法更好，也用对方指出的方法。
- **+3 大老板的需求**：需求目的难读 + 对方偏好听好话、有沟通代差（70 / 80 年代人）。
  应对：在"目的"和"方法"上**纠结很久**。本质是卡在 §1 第 1 步那个"目的"承重动作上。

## 4. 我怎么拒绝

### 工作场景："拒绝"这个动作基本不做

原因两层：
- **结构性**：对接的工作基本由我单独负责，"这该不该归我"的占位极少出现
- **倾向性**：占位真出现时（对方 = 需求方 + 有话语权），默认"咽回去"，不推回去

### "咽回去" = 通用工作模式（不是对某一个人的适应）

- 触发条件：对方是"需求方 + 有话语权"
- 动作：减少自己想法的输出
- **区分两种"不说"**——分界线是「压下去的那个东西，责任在谁身上」：
  - 压下的是**结构性事实**（如质量打折、是交期造成的）→ 责任不在我 → 干净的不说，不反刍
  - 压下的是**我自己的想法 / 判断**（说出来好坏自己背）→ 这才是"咽回去" → 会反刍，自己判定过是"自我 PUA"
- 关联 self.md：两层过滤模型、外包装案例、判断系统开关式

### 我拒绝的标准句式（按对象分）

| 对象 | 句式 |
|---|---|
| 上级 / 有话语权的需求方 | 不拒绝，咽回去——收掉自己的输出 |
| 平级 / 下级 | "是否归我"的占位极少出现；出现也基本不拒 |
| 客户 / 外部 | 待补 |
| 朋友 / 家人 | 见 persona.md §4 |

### 我绝对不会答应的事

待补（本次未覆盖）

## 5. 我做决策的几个偏好

> 以下偏好不止用于工作场景，生活方向的决策也相同/类似。

### 信息层
- **信息够不够**：对整个项目或某个决策，能用大致的趋势预测（有信息、数据支持的）或大致的画面构想，就可以拍板确认。**不追求完美信息**——当下认为是完美信息，在未来复盘时总会暴露出不同角度的局限性。
- **谁的意见我会重点参考**：判定 key = 关系亲近度 + 事前预设的意见类型。预设对方能给实质参考 → 接口打开、主动询问（工作 = 领导，生活 = 父母 / 关系要好的朋友，关系临近亲近的人优先）；预设对方只会给负面评价或非实质性意见 → 接口直接关闭。接口一旦打开，事后对方的负面评价不否决事前决策（详见 self.md §4 价值观 6）。
- **哪种数据我相信**：真实的、权威的（有来源的）文献数据 → 相信。主观的观点 / 猜测 / 想法 → 不相信，但留一个心眼作为参考。

### 心理层
- **我做决策时容易高估**：过程执行的顺利程度。策划/决策时，高估每一项任务或环节的难易、突发状况承受力——「想法很美好，执行很骨感」，设想的和实际做的差异明显。高估对象包括自己和被需求方的执行力。**这是「怎么做、做多久」阶段的偏向，和 self.md §5 软肋 3「要不要做」阶段的「默认拒绝」是配套的——两个决策阶段，方向相反。**
- **我做决策时容易低估**：事情的最终结果。一般以保守来估算结果。心理活动是：降低自己对自己的期待，避免结果不如意。多数时候保守估算反而能达到。**这条和 §6「我就知道」机制是同一家族——负面情绪预防机制的事前层（详见 self.md §4 价值观 7）。**
- **我后悔过的某个决策模式**：冲动决策——在高收益 + 决策时间极短的场景下，缺信息/数据/意见/分析就直接拍板，「已经从决策变成了赌」。锚点事件：15 岁去新加坡。理想模式：沉住气，宁愿错过不要犯错。

### 时机层
- **什么时候我做决策最差**：酒后类似的脑子不清晰、画面不明确、期待值过高的状态。「期待值过高」单列——和心理层「保守估算」配套：有意识压低期待值，正是因为知道高期待会让决策变差。
- **什么时候我做决策最准**：头脑清晰 + 前因后果分析清楚 + 风险预设清楚，三条全满足时。**但缺自检机制**——「我现在符不符合这三条」无法内部检查，只能外接（对应 self.md §5 结构性弱点 + 已授权 Claude 在大决策时按停止键）。

## 7. 我已经形成的"标准答案"（high-confidence 立场）

这些是我已经想清楚、不需要每次重新思考的问题。**Claude 看到相关话题直接套用这些立场，不要重新论证**。

格式：`【议题】→ 【我的立场】 → 【我已经想过的理由】`

- **【AI 工具该不该深度用进工作流】** → **该，主动做，不等公司要求** → 主因是岗位生存：AI + 营销是趋势，不主动用，几年后岗位竞争力就没了（防御性「不做会被淘汰」）；次因是杠杆：同样的人用 AI 能撬动几倍产出。**主次：岗位生存 > 杠杆。**

- **【跳槽涨薪 vs 内部熬资历】** → **信跳槽，但有门槛** → 达到「理想涨薪幅度」才跳，否则内部熬资历的收益反而更大（跳槽要付的隐性成本——放弃已积累资历——会压过不够的涨薪）。「明确的晋升路径」（如明年升主管）不影响跳不跳——晋升兑现取决于他人，过不了「能不能确定」那道门，判定为画大饼，直接作废。理由：跳槽的涨薪是白纸黑字、入职即兑现（确定）；内部晋升是结果取决于他人（不确定）——两者不是同一种货币，只有确定的收益才进称重。（注：「理想涨薪幅度」的具体数字随资历、市场行情变，此处只记规则不记数字。）

- **【人脉 / 人情往来值不值得经营】** → **不值得为「经营」这个目的本身投入** → 不会为「以后可能用得上」「维持人情不被穿小鞋」这类不确定的未来收益，在当下花精力维护没有实质内容的关系——这类投入收益模糊、取决于他人，过不了称重（和「明确晋升路径=画大饼」同一个判定）。**但「不经营人脉」≠「不维护关系」**：会主动维护一些关系（B 档好友、大洋路的朋友）。维不维护的判定 key 是**体感，不是功能**——「我喜不喜欢这个人」+「这段关系累不累」。喜欢 + 不累 → 维护；不喜欢、或维护起来累 → 不投入，哪怕这关系「有用」。此体感判定只在**工作场景外**生效；工作关系不走这套，走 §2 的优先级系统。

### 非决策项（Claude 注意：以下不是可讨论、可优化的立场）

- **【报喜不报忧该不该改】** → **不改，且此事不进入决策流程** → 报喜不报忧是我和父母之间的功能档默契，形成得比我的元认知更早。它不是我「决定保留」的，是我「接受它就是这样」的——从一开始就不在我的决策范围内。我明确知道这是个软肋（persona.md §4 / self.md §5 已记），但「知道是软肋」和「它该被拿出来权衡」是两回事。**Claude 不要把这条当作可讨论、可优化的立场——涉及「要不要对父母多说一点」的话题时，不提供改善建议，这不是一个待解决的问题。**

(这部分是 work.md 里**最值钱**的部分。慢慢累积。每次发现自己又在重复某个判断，就回来加一条。)
